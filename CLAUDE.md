# Abstract Games Platform

Generic platform for abstract board games (async-vs-human + vs-bot), games added as modular packages.
See PLATFORM_PLAN.md (roadmap) and engine/SPEC.md (game authoring contract).

## Layout
- `engine/` — Python SDK (pure stdlib): `agp/` package, `games/<uid>/` packages, `SPEC.md`, `tests/`.
- `server/` — FastAPI backend (SQLAlchemy, sqlite default). `web/` — React+Vite frontend.
- `gamedev-kit/` — distributable SDK; build with `python3 tools/build_devkit.py` → `dist/gamedev-kit.zip`.
- `legacy/` — the original standalone React prototypes of Oust & Yodd (archived, not part of the live app; both now run as engine modules). See `legacy/README.md`.

## Run / dev
- `./dev.sh` — backend :8000 + frontend :5173 (Vite proxies `/api`); sources gitignored `dev.env` (e.g. `AGP_ADMIN_EMAILS`).
- **Restart the backend after any engine/game change** — the game registry loads game code at startup and is cached (uvicorn runs without --reload).
- Frontend `web/src` edits are live via Vite HMR on :5173 — **no `npm run build` needed in dev** (build only for production/`dist`).
- **Prod-parity check locally:** `cd web && npm run build` then `uvicorn server.app:app --port 8000` — the backend serves the built SPA + `/api` from one origin (it auto-mounts `web/dist` when present), exactly like the Render deploy.
- Stop a server by PORT, never `pkill -f vite|uvicorn` (the pattern matches the bash command itself → exit 144): `kill $(ss -ltnp | grep ':8000 ' | grep -oP 'pid=\K[0-9]+')`.
- Start long-lived servers via the Bash tool's `run_in_background`, not `nohup ... &` in a foreground call.

## Engine / authoring a game
- A game = `engine/games/<uid>/{manifest.json, game.py}` (game.py = one `agp.game.Game` subclass; uid comes from manifest — don't hardcode it in the class).
- Validate/iterate: `cd engine && PYTHONPATH=. python3 -m agp.cli validate|playtest|render|pack games/<uid>`.
- Moves are strings: `>`-separated cell-id path (`"2,1>3,4>5,6"`), optional `=CHOICE` suffix (promotion). Cells: `"col,row"` square, `"q,r"` axial hex.
- **Guarantee termination** (conformance plays random games to a terminal): add no-progress + hard-ply-cap draw rules for games that could loop.
- Optional hooks: `describe_move` (move-log notation), manifest `category` (lobby grouping). "Win as event" games store the result in state (e.g. Oust/chess `winner`), not inferred from the board.
- **Scored games: a genuine tie is an honest DRAW** (`winner=None` / `returns [0,0]`), NEVER a fabricated tiebreak "that should be unreachable" — it usually IS reachable (e.g. an early symmetric double-pass) and a fake winner is an exploitable bug. "Draws are impossible" claims usually mean *full* boards; still handle the reachable tie.
- Rule variants / board sizes → manifest `options` (auto-rendered as dropdowns); make a separate package only for a genuinely distinct game.
- Ship a **`rules.md`** (one page, rules *as implemented* — the local source of truth) in each game package. The web UI shows it via a "Rules" button and serves it at `/api/games/<uid>/rules`; `manifest.bgg_url` becomes the "official source" link. Supported Markdown: headings, `**bold**`/`*italic*`/`` `code` ``, `-`/`1.` lists, `[links](url)`.
- The lobby/quick-play game selector is a searchable, category-grouped `GamePicker` (`web/src/`); `manifest.category` and `tags` feed grouping and search.
- Non-cell legal moves (e.g. `"swap"`, `"pass"`) auto-render as action buttons; `=CHOICE` moves show a picker.
- **Chess-family games subclass `agp.chesslike.ChessLike`** (board model, slider/leaper move-gen, attack/check, draws, serialize/render). A variant just sets `WIDTH/HEIGHT`, a `PIECES` movement table (`{letter: (slide_dirs, leap_offsets)}`), `HEAVY` material, `setup_board()`, and four strategies — `PAWN` (`StandardPawn`/`BerolinaPawn`), `PROMOTION` (`LastRankPromotion`/`GrandPromotion`), `CASTLING` (`StandardCastling`/`NoCastling`), and `DROPS` (`NoDrops` default / `CrazyhouseDrops`). See `games/{chess,berolina,grand_chess,los_alamos_chess}` (~40 lines each). Validate new variants with a perft check against known node counts.
- **Drops / off-board reserve** (the first UI capability investment, shipped via Crazyhouse): the `DROPS` strategy adds a per-seat `hands` reserve + a `promoted`-square set to `CState` (both no-op/absent unless `DROPS.enabled`, so the other ~20 variants are byte-identical). A drop move is the string `"L@c,r"`; captures bank into the reserve; `render()` emits `spec.reserve = {seat: {letter: count}}`. **No server change** — the enforced path is `move in legal_moves`. The web `Board.jsx` shows two reserve trays (seat 1 top / seat 0 bottom): click your chip → legal drop targets highlight → click to drop. Reuse this for Shogi (its own `DropRules` for drop-zone/nifu) and the Morris family (different board, custom adjacency). Anchor with python-chess `CrazyhouseBoard` (see `games/crazyhouse/_diff_pychess.py`, manual/one-time).
- **Randomness without a chance node:** roll/deal in `apply_move`/`initial_state` and STORE the outcome in state (the die / dealt cards); set manifest `has_randomness: true`. The generic UI/bot then need no CHANCE handling (EinStein, Onitama). `num_players` is a fixed property read before any state exists, so it can NOT be a manifest option — a variable-player game picks one count (Chinese Checkers is fixed 6-seat).

## Server
- `DATABASE_URL` (default `sqlite:///./agp.db`). Uploads run game code IN-PROCESS (RCE) → gated, closed by default: `AGP_ADMIN_EMAILS` allowlist or `AGP_ALLOW_OPEN_UPLOADS=true`. Real sandbox is deferred.
- Game registry caches at startup; `POST /api/games/upload` hot-reloads it.
- **Schema changes: NO Alembic — `db.init_db()` runs `Base.metadata.create_all` only.** It creates NEW tables but NEVER ALTERs existing ones, so adding a column to `User`/`Match` won't appear on the live Neon DB. Add a NEW table instead (how ratings/deadlines/chat were done), or compute from existing fields (move deadlines derive from `match.updated_at`).
- **Match-completion hooks** (`server/games.py`): `rate_finished_match(db, match)` (Glicko-2, `server/rating.py`) + `forfeit_if_overdue`/`sweep_overdue` (move deadlines) are idempotent and called after `/move`,`/resign`,lobby-load. Ratings/leaderboards/profiles/replay/chat/spectating/quick-pair all live; `match_view` attaches per-seat rating+delta+deadline.

## Known issues — see `KNOWN_ISSUES.md`
- **✅ MCTS bot speed — FIXED (2026-06-26).** `MCTSBot` now takes a `max_time` wall-clock budget (server passes `AGP_BOT_MAX_TIME`, default 3s) + `max_rollout=50` + an optional `game.heuristic(state)` eval at the rollout cutoff (`ChessLike.heuristic` = tanh material balance; draw fallback for games without it). Chess move minutes→~3s. Add a `heuristic` to a `Game` to make its bot stronger — but it **MUST return a LIST of `num_players` payoffs** (same convention as `returns`); a bare float raises `TypeError: 'float' object is not subscriptable` in back-prop (`payoffs[p]`). **Only fires when the rollout cutoff is reached**, so a game shorter than `max_rollout` hides the bug (blokus_duo shipped it) — test with a low `max_rollout=4` to force it. See SPEC.md. Free-tier knob: raise `AGP_BOT_MAX_TIME` env if the bot feels weak.

## Adding games at scale (the "game factory")
- The bundled library (~135 games) was largely built by an autonomous loop. `GAME_BACKLOG.md` is the capability map (the abstract-game universe bucketed by the platform primitive each needs); `GAMES_QUEUE.md` is the live status + the merge gate + escalation digest. Read both before a bulk game effort.
- **Session hygiene — clear between waves, don't carry history.** Waves are independent and ALL durable state lives on disk (git commit messages, `GAMES_QUEUE.md` SESSION HANDOFF, `GAME_STATUS.md`, the `game-factory-*`/`platform-direction` memories, this file). So at a **clean checkpoint** (wave committed + pushed + suite green + queue/memory updated + tree clean) a `/clear` loses nothing durable and resets the token cost; a fresh session re-orients in ~2-3 reads (memory auto-loads → read the queue handoff → go). Prefer `/clear` over `/compact` at wave boundaries (compact keeps a summary you don't need); use `/compact` only MID-wave (agents in flight / uncommitted decisions). **Before clearing, PRE-STAGE the next wave in the queue handoff** — the specific games, deduped, with sourced rules — so the next session is a one-line kickoff ("continue the game factory — next wave per the GAMES_QUEUE handoff") and doesn't re-scout.
- The factory is a reusable dynamic **Workflow** (script under the session's `workflows/scripts/game-factory-*.js`): per game, one agent implements the package and a *different* agent adversarially verifies it; the orchestrator owns a deterministic merge gate (auto-merge on a published anchor or a clean independent review; queue only genuine ruleset decisions / new board shapes). See the `game-factory-loop` memory.
- **Anchor chess-family games with perft**; for variants, **python-chess / shakmaty** (install in `.venv`) are gold-standard differential oracles — use them for one-time verification only. **`pyffish` (Fairy-Stockfish bindings, pip-install in `.venv`) is a general oracle for ANY FSF variant** — built-ins (duck, shatar, shogun via `[shogun:crazyhouse]`) and custom rulesets via `pyffish.set_option`/`load_variant_config` on a `variants.ini` block; perft + legal-move differential. (Calling an unloaded custom variant SEGFAULTs — load the config first.) For large shogi, **`hachu` (Debian `apt-get download hachu`, the 0.21 release — the github 0.23 build is buggy)** speaks CECP and differentials Chu/Dai; `@@@@` is its Lion-pass null-move, and feeding an invalid usermove makes it dump its full root move list. **`pentobi` (GPL, github.com/enz/pentobi) is the reference engine for the WHOLE Blokus family** (classic/duo/trigon/junior/nexos/callisto/gembloq): `cmake -B build -DPENTOBI_BUILD_GTP=ON -DPENTOBI_BUILD_GUI=OFF` → `pentobi-gtp` (needs no Qt; **`cmake` is NOT installed and apt needs sudo → `.venv/bin/pip install cmake`**). GTP: `set_game blokus duo` (the FULL name — `duo` alone is rejected), `all_legal <colour>` (duo=`b`/`w`, 4p=`1`..`4`; **multi-line response ended by a blank line** — reading one line silently under-reports), `play`, `final_score` (`0` = draw). Its source is authoritative for geometry/bonuses. Differentials live at `games/{blokus,blokus_duo,blokus_trigon}/_diff_pentobi.py` (env `PENTOBI_GTP`); compare **covered-CELL-SETS, not move strings**, so they survive move-notation changes.
- **A SOLVED game is the strongest anchor there is:** if the state space is ≤~1e9 or a published solved value exists, write a one-time full solver (retrograde or memoized negamax) and assert the designer's published game value — this validates the ENTIRE ruleset end-to-end at once (done for Mijnlieff = 2nd-player win via 2.47B-node alpha-beta, Neutreeko = draw / 51-ply / 3.08%-neutral via a 3.4M-position retrograde solve, Soluna, Bobail). One-time (not in the pure-stdlib selftest); freeze the result.
- **Build/QA agents' briefs are often wrong on metadata** (BGG ids, designer/publisher/year): tell every agent "sources override the brief" — they corrected me on Omega/Veletas/Qawale/Neutreeko/Zola this run. Publisher rulebook PDFs are authoritative and reliably fetchable: **nestorgames.com/rulebooks/<NAME>_EN.pdf**, **marksteeregames.com/<Name>_rules.pdf** (MOST Steere sheets use the `_rules` suffix — a few, e.g. `Zola.pdf`, omit it; probe the filename with `curl -o /dev/null -w %{http_code}`. They're Illustrator vector PDFs with NO text layer → `pdftoppm -png -r 200` and READ the figures, which carry setup + worked examples), Cosmoludo/Gigamic/Blue-Orange sheets, spielstein.com (server-rendered), and **BGA's `en.doc.boardgamearena.com/Gamehelp<game>`**. BGG's XML API now 401s — use `api.geekdo.com/api/geekitems`, Wayback, or web search.
- **Dedup every candidate first** (`ls engine/games/` + `git cat-file -e HEAD:engine/games/<uid>/game.py`) — the library is 200+ games; most famous abstracts already exist. Screen MECHANICAL clones too (read the closest existing game's `rules.md`), not just name collisions.
- **Build agents must verify rules vs authoritative sources and OVERRIDE the brief on any conflict** — the quality backbone (agents corrected briefs ~14× in one run, e.g. Judkins Shogi's back-rank order).
- **A build agent's package files land on disk (and `validate`) BEFORE its completion notification** — QA as soon as `games/<uid>/{game.py,manifest.json}` exist; don't wait for the agent to report.
- **Model-cap ops:** override an agent's model via the Agent tool `model:` (`opus`↔`fable`) when one caps; BOTH can cap at once (a hard stop — stage briefs, relaunch after reset). A killed agent leaves an EMPTY dir (`rmdir`) or a partial package (`rm -rf`) → clean + relaunch fresh; keep a partial ONLY if all 4 files exist AND validate+selftest pass. Build agents occasionally drop scratch files in the repo ROOT — `git status` after a wave and clean them before committing.
- **Bulk throughput:** ~3 build agents in parallel; FAST-PATH perft-anchored ChessLike/ShogiLike variants (standard render ⇒ ~nil render-bug risk → commit on the anchor + a `render()` shape probe); BROWSER-VERIFY only custom-render games; BATCH the backend-restart + browser smoke-checks (don't restart per game).
- **Need a NEW `Board.jsx` render primitive?** The orchestrator owns `web/src/Board.jsx` (a render bug white-screens EVERY game): write the opt-in primitive + REGRESSION-CHECK an existing square + cards (Onitama) + polygons game, document the contract in `SPEC.md`, THEN a build agent writes the engine to emit it. Gotcha: on `polygons` boards the inscribed radius `s.r` ≠ half-width — tile/track/notch edge-glyphs use the cell's `hw`/`hh` bounding half-extents to sit on the border.

## Testing
- Engine: `cd engine && PYTHONPATH=. python3 tests/test_games.py`.
- **Per-game `selftest.py`**: a game package may ship `games/<uid>/selftest.py` — a standalone script asserting its correctness anchor (perft / rule positions). `tests/test_games.py::test_package_selftests` runs every one. **Selftests MUST be pure-stdlib** (import only `agp` + their own game; no `python-chess`/numpy) and fast — the suite runs them under system `python3` where pip-only deps are absent.
- **Probe a package outside the CLI:** `from agp.loader import load_from_dir; man, g = load_from_dir(Path("games/<uid>"))` — game classes take NO ctor args (uid comes from the manifest), so instantiating the class directly fails with "takes no arguments". If a board needs a one-off library to build (e.g. PIL image-segmentation), do it OFFLINE and ship the result as `board.json` loaded at runtime (see `games/pex`) so `game.py`/`selftest.py` stay pure-stdlib.
- **Don't trust a piped suite exit code:** `python3 tests/test_games.py | tail` returns the pipe's (tail's) exit, always 0 — run it unpiped to a file with `echo SUITE_EXIT=$?`, or grep for `all tests passed`.
- **The full suite is SLOW** (~5 min at 200+ games). Per-game `validate` + `selftest` is the fast gate; run the full `tests/test_games.py` only as a periodic backstop (unpiped; grep `all tests passed`).
- **Don't run package-scanning tools while a build agent is mid-write** — `gen_game_status.py`, the full suite, and a backend restart all break or load partials on a half-written package (missing `manifest.json` → `PackageError`). Run them only when builds are quiet.
- **"Win as event" + selftests:** `winner` is set only inside `apply_move`, so `is_terminal` is False on a hand-built no-move/stuck position. Test stuck/annihilation wins by REACHING them via `apply_move`, not by constructing the dead state.
- **Restart the backend to pick up a NEW game** (don't trust auto-reload for new packages). Kill STRICTLY by port; do NOT `ps`/`grep`/`pkill` for `uvicorn server.app` — your own restart command's line contains that string, so the match kills your shell (exit 143/144). Then start uvicorn via `run_in_background` with no grep.
- Backend via `TestClient`: call `server.db.init_db()` first (bare TestClient skips the startup lifespan → no tables). Use `httpx` in `.venv` (not `requests`).
- **RenderSpec-format bugs are INVISIBLE to `validate`/selftest** (those test the Python game logic, not `Board.jsx`) — a 100%-logic-correct game can still white-screen-crash the board. **Browser-verify EVERY new game** (a 0-nodes / blank page after START = a render crash; the lobby still renders, so it's game-specific). Diagnose via `render(initial_state())['board']` vs `web/src/Board.jsx`.
- Browser checks use pinchtab: click custom buttons by **ref**; click SVG board cells via `[data-cell="c,r"]` CSS and non-a11y SVG elements (wall ghosts, cards) via a class selector with `--mode dom`; coordinate clicks are flaky — verify with a screenshot.
- pinchtab session: `export PINCHTAB_SESSION=$(pinchtab session create --agent-id <id> | tail -1)` — all commands then share that tab. Read the move-log/DOM with `pinchtab eval --json '<js>'` (a screenshot goes byte-stale right after a click → re-shoot or use eval). Selecting a game card: search substrings match many cards, so pick by exact title over `button.game-card-main` via eval, not blind index. **Sessions expire on long runs** — recreate with `pinchtab session create` on a `401 bad_session`. **In Quick Play you must click the game CARD to select it BEFORE hotseat/START**, else it launches the default game (search by a unique term — e.g. "Baduk" for Go).
- **Stale-bundle false alarm (bit 3× in one session):** the browser caches `index.html` pointing at an OLD hashed bundle, so a NEW frontend feature looks completely broken (e.g. palette trays absent, every move rendering as an action button) while the code is fine. Check what actually loaded — `pinchtab eval 'performance.getEntriesByType("resource").filter(e=>e.name.includes(".js")).map(e=>e.name.split("/").pop())'` — vs `ls web/dist/assets/*.js`; fix with a cache-busting `?cb=$RANDOM` nav. Suspect this BEFORE suspecting the game. (`npm run build` first: uvicorn serves `web/dist`, mounted at startup.)
- **Simulating hover for React:** `el.dispatchEvent(new MouseEvent("mouseover",{bubbles:true}))` triggers `onMouseEnter` — but adding `relatedTarget` breaks it (React reads it as movement *within* the element), and reading the DOM in the SAME `eval` sees the pre-render state. Dispatch and assert in SEPARATE `eval` calls.
- **pinchtab IDPI self-false-positive:** the injection scanner FALSE-POSITIVES on our OWN lobby text (game descriptions trip jailbreak/urgency heuristics) → `snap`/`text`/`find` return 403/timeout. Use `screenshot` (pixels, bypasses the content scanner) for visual checks + CSS/`text:` selectors for clicks. The instance is also flaky (`context deadline exceeded` → just retry). No hotseat deep-link — only `?match=<id>`; reach a board via QuickPlay (click the CARD) or log in + open a match.
- **pinchtab external sites:** navigation is gated by an allowlist (`~/.pinchtab/config.json` → `security.allowedDomains`; `403 IDPI: Domain not in allowlist` if missing). The hosted apps (`abstract-games.onrender.com`, `gamefinder-seven.vercel.app`, `generic-poker.onrender.com`, `erikbar.duckdns.org`) are allowlisted for UI testing — `pinchtab server restart` after editing the list. Otherwise verify a live site with `curl`.

## Conventions
- Frontend: one generic `Board` renderer; all board types (`square`/`hex`/`polygons`) go through a unified polygon shape model — `polygons` games supply each cell's vertices as **`board.cells` = a LIST of `{id, points}`** (NOT a dict; vertex key is `points` not `polygon` — getting it wrong crashes the renderer; see SPEC.md). No per-game UI. Seat colors in `web/src/colors.js`.
- **RenderSpec capabilities (opt-in, generic; all 7 backlog UI gaps now shipped — see `GAME_STATUS.md`):** `reserve` trays + click-to-drop (move `"L@c,r"`); **>2-seat** (6 colours in `colors.js`; QuickPlay seats `view.num_players` — MCTS already backs up per-player payoffs, no engine change); `board.lines` (under-cell grooves) & `board.overlay` (over-cell lines, e.g. TwixT bridges), both in cell-coord space; `board.tints` (`{cellId: colour}` terrain/edges); `board.walls` (`{h,v}` groove slots → ghost-click placement, Quoridor); `piece.stack: [owner,…]` (tower glyph, Lasca); `board.cards` (movement-pattern strip, Onitama); **`palette`** = multi-cell POLYOMINO tile-laying (move `"KEY:o@c,r"` = drop syntax + orientation index; per-seat trays, orientation strip, ghost footprint; `palette.shared` = one "Pool" tray; `grid:"tri"`+`parity[]` for polyiamonds — Blokus/Trigon/Pentominoes/Cathedral) and **`piece.shape:"fill"`** (flood the whole cell so a tile reads as one block, not a disc per cell). **`palette` anchor contract: every orientation MUST contain `[0,0]`** (anchor = bottom-most then left-most COVERED cell; NEVER the bbox corner — a plus-pentomino's corner is empty) ⇒ negative `dc`, so placement loops must bound BOTH sides (`range(-lo_c, W-hi_c)`); the naive `range(W-max)` silently DROPS placements. `GAME_STATUS.md` (via `engine/tools/gen_game_status.py`) is the live per-game catalogue — regenerate after adding a game.
- Porting a Zillions of Games (.zrf) game: use the `zillions-to-platform` skill (`.claude/skills/`).
- Git: solo project, commit directly to `main`.
